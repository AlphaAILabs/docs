## 部署指南

### 1. 登录服务器并下载程序

```bash
# 登录服务器
ssh user@your-server

# 下载程序（以 Linux x64 为例）
wget https://github.com/AlphaAILabs/variational-turbo/releases/download/v1.0.0/variational-turbo-linux-amd64

# 添加执行权限
chmod +x variational-turbo-linux-amd64
```

<Info>
  其他版本下载地址：https://github.com/AlphaAILabs/variational-turbo/releases
</Info>

### 2. 使用 tmux 启动

```bash
# 创建 tmux 会话
tmux new -s turbo

# 启动程序
./variational-turbo-linux-amd64

# 按 Ctrl+B 然后按 D 分离会话（程序继续后台运行）
```

<Tip>
  **tmux 常用命令**：
  - `tmux attach -t turbo` - 重新连接会话
  - `tmux kill-session -t turbo` - 终止会话
</Tip>

### 3. 配置 Caddy 反向代理

编辑 `/etc/caddy/Caddyfile`：

**使用域名（自动 HTTPS）：**

```
your-domain.com {
    reverse_proxy localhost:8080
}
```

**使用 IP（自签名证书）：**

```
https://YOUR_SERVER_IP:443 {
    reverse_proxy localhost:8080
    tls internal
}
```

将 `YOUR_SERVER_IP` 替换为您的服务器公网 IP。

重启 Caddy：

```bash
sudo systemctl restart caddy
```

**开放防火墙端口：**

```bash
# Ubuntu/Debian (ufw)
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# CentOS/RHEL (firewalld)
sudo firewall-cmd --permanent --add-port=80/tcp
sudo firewall-cmd --permanent --add-port=443/tcp
sudo firewall-cmd --reload
```

<Warning>
  如果使用云服务器（如阿里云、腾讯云、AWS 等），还需要在**云控制台的安全组**中放行 80 和 443 端口。
</Warning>

<Warning>
  使用自签名证书时，浏览器会显示安全警告，点击"高级" → "继续访问"即可。Chrome 也可以直接输入 `thisisunsafe` 跳过。
</Warning>

### 4. 访问并配置

1. 浏览器访问 `https://your-domain.com` 或 `https://your-server-ip`
2. 使用 AlphaLabs 官网注册的 **EVM 地址** 和 **客户端密码** 登录
3. 点击"添加账户组"，配置主账户和对冲账户的 Token 和 Base URL
4. 配置策略参数后点击"启动"

<Info>
  获取 Variational API 配置请参考 [Variational 平台配置](/platforms/variational-setup#api-配置)
</Info>

<Warning>
  **重要提醒**：策略运行中**请勿刷新页面**，否则需要重新启动策略。
</Warning>
