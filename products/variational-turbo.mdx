## 部署指南

### 1. 登录服务器并下载程序

```bash
# 登录服务器
ssh user@your-server

# 下载程序（以 Linux x64 为例）
wget https://github.com/AlphaAILabs/variational-api-service/releases/download/v1.0.0/variational-service-linux-amd64

# 添加执行权限
chmod +x variational-service-linux-amd64
```

<Info>
  其他版本下载地址：https://github.com/AlphaAILabs/variational-api-service/releases
</Info>

### 2. 使用 tmux 启动

```bash
# 创建 tmux 会话
tmux new -s turbo

# 启动程序
./variational-service-linux-amd64

# 按 Ctrl+B 然后按 D 分离会话（程序继续后台运行）
```

<Tip>
  **tmux 常用命令**：
  - `tmux attach -t turbo` - 重新连接会话
  - `tmux kill-session -t turbo` - 终止会话
</Tip>

### 3. 配置 Caddy 反向代理

vim `/etc/caddy/Caddyfile`：

**方式一：使用域名（推荐）**

如果您有域名，Caddy 会自动申请 Let's Encrypt 免费证书，浏览器访问时显示安全连接：

```
your-domain.com {
    reverse_proxy localhost:3000
}
```

**方式二：使用 IP（自签名证书）**

如果没有域名，可以使用 IP 直接访问。由于 IP 地址无法申请正规 SSL 证书，Caddy 会使用自签名证书：

```
https://YOUR_SERVER_IP:443 {
    reverse_proxy localhost:3000
    tls internal
}
```

将 `YOUR_SERVER_IP` 替换为您的服务器公网 IP。

<Info>
  **关于证书的说明**：
  - **域名访问**：Caddy 自动申请 Let's Encrypt 证书，浏览器显示"安全"
  - **IP 访问**：使用自签名证书，浏览器会显示"不安全"警告，但数据传输仍然是加密的，点击"继续访问"即可正常使用
</Info>

重启 Caddy：

```bash
sudo systemctl restart caddy
```

**开放防火墙端口：**

```bash
# Ubuntu/Debian (ufw)
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# CentOS/RHEL (firewalld)
sudo firewall-cmd --permanent --add-port=80/tcp
sudo firewall-cmd --permanent --add-port=443/tcp
sudo firewall-cmd --reload
```

<Warning>
  使用自签名证书时，浏览器会显示安全警告，点击"高级" → "继续访问"即可。Chrome 也可以直接输入 `thisisunsafe` 跳过。
</Warning>

### 4. 访问并配置

1. 浏览器访问 `https://your-domain.com` 或 `https://your-server-ip`
2. 使用 AlphaLabs 官网注册的 **EVM 地址** 和 **客户端密码** 登录
3. 点击"添加账户组"，配置主账户和对冲账户的 Token 和 Base URL
4. 配置策略参数后点击"启动"

<Info>
  获取 Variational API 配置请参考 [Variational 平台配置](/platforms/variational-setup#api-配置)
</Info>

<Warning>
  **重要提醒**：策略运行中**请勿刷新页面**，否则需要重新启动策略。
</Warning>
